# **Sweet Shop / Coursework 2025 DB**

## **Загальна інформація**

**Даний проект являє собою інтернет магазин будь-яких товарів, будь-якої категорії** (без повної реалізації, у даному випадку обрана категорія товарів - солодощі).

**Члени команди:** Архіпенко Артємій ІМ-43

## **Технологічний стек**

**Мова програмування:** C#

**ORM:** Entity Framework (Npgsql)

**Фреймворк тестування:** xUnit.net

# **Інструкції з налаштування**

**1. Клонуйте репозиторій.**

**2. Відкрийте термінал на найвищому рівні проекту.**

**3. У терміналі пропишіть команди:**

```
docker compose build
```

**Після повідомлення у терміналі про те, що образ збілдився, пропишіть наступну команду:**

```
docker compose up
```

**4. Відкрийте Ваш браузер, далі перейдіть за посиланням "http://localost:8080".**

**5. Для доступу до "Адмін панелі" Вам необхідно залогінитись під користувачем з вказаними даними: Login - admin@gmail.com , Password - 12345678 .**

## **Проведення тестів**

**1. Відкрийте термінал у теці COURSEWORK\IntegrationTests .**

**2. Пропишіть команду:**

```
dotnet test
```

# **Структура проєкту**

**На найвищому рівні проекту знаходяться такі теки як "src", "docs" та "IntegrationTests".**

**src** - тека з основним проектом.

**docs** - тека зі схемою бази даних і запитами до неї.

**IntegrationTests** - тека з проектом тесту та відповідно тестами до основного проекту.

### **Розглянемо теку "src" більш детально**

**Всередині даної теки можна виділити такі розділи:**

**Controllers** - контролери для обробки запитів.

**DTOs** - об'єкти передачі даних. Можна використовувати, наприклад, щоб запити реєстрації або авторизації не містили створених у системі хеш-паролей.

**Data** - контекст бази даних та Seed дані (Admin користувач). За допомогою "ApplicationDbContext.cs" Entity Framework може зв'язати нашу систему з базою даних PostgreSQL, а "DbInitializer.cs" надає можливість при запуску контейнерів проекту у будь-якому середовищі, одразу у "чистій" базі даних створювати користувача з правами Адміна.

**Migrations** - міграції бази даних.

**Models** - моделі таблиць у базі даних.

**Services** - сервіси, які реалізують логіку проєкту.

**Views** - представлення, які відображаються користувачу сайту.

## **Приклади API/використання**

**У проєкті реалізовано такий функціонал:**

**1. Авторизація користувача.**

**2. Головна сторінка з каталогами та продуктами.**

**3. Корзина, у якій можна змінити кількість продуктів та перейти до замовлення.**

**4. Оформлення замовлення (без вказання адреси).**

**5. Профіль з карткою користувача та історією замовлень.**

**6. Адмін панель у відповідного користувача, у якій можливо керувати за категоріями, продуктами та замовленнями.**
